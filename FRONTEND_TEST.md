# フロントエンドTodo管理画面の確認方法

## アクセス方法

1. **フロントエンド**: http://localhost:3000
2. **Todo管理画面**: http://localhost:3000/todos
3. **バックエンドAPI**: http://localhost:8000/api/v1/todos/
4. **APIドキュメント**: http://localhost:8000/docs

## 動作確認手順

### 1. フロントエンドにアクセス
ブラウザで `http://localhost:3000` を開きます。

### 2. Todo管理画面に移動
ナビゲーションバーから「Todos」をクリックするか、直接 `http://localhost:3000/todos` にアクセスします。

### 3. 機能確認

#### Todo一覧の表示
- ページ読み込み時に、既存のTodoが自動的に表示されます
- Todoの件数が表示されます

#### Todoの作成
1. 「新しいTodoを作成」セクションでタイトルを入力
2. （任意）説明を入力
3. 「作成」ボタンをクリック
4. 作成されたTodoが一覧に追加されます

#### Todoの完了/未完了の切り替え
- 各Todoのチェックボックスをクリックして完了状態を切り替えられます
- 完了したTodoは視覚的に区別されます（グレーアウト、取り消し線）

#### Todoの削除
- 各Todoの「削除」ボタンをクリック
- 確認ダイアログが表示されます
- 「OK」をクリックするとTodoが削除されます

## 現在のデータ

バックエンドには以下のTodoが保存されています：
- 最初のTodo（動作確認用）
- テストTodo（動作確認）
- 動作確認（テスト）

## トラブルシューティング

### CORSエラーが発生する場合
バックエンドのCORS設定を確認してください。`app/main.py`で以下の設定が有効になっています：
```python
allow_origins=["http://localhost:3000", "http://127.0.0.1:3000"]
```

### API接続エラーが発生する場合
1. バックエンドが起動しているか確認: `docker compose ps`
2. バックエンドのログを確認: `docker compose logs backend`
3. APIエンドポイントを直接確認: `curl http://localhost:8000/api/v1/todos/`

### フロントエンドが表示されない場合
1. フロントエンドが起動しているか確認: `ps aux | grep vite`
2. ポート3000が使用可能か確認: `lsof -i :3000`
3. 必要に応じて再起動: `cd frontend && npm run dev`

## 開発者ツールでの確認

ブラウザの開発者ツール（F12）で以下を確認できます：
- **Networkタブ**: APIリクエスト/レスポンスの確認
- **Consoleタブ**: JavaScriptエラーの確認
- **Applicationタブ**: ローカルストレージやセッションの確認
